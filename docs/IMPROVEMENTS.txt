Medium priority (v2 or v3)
- OpenAPI/Swagger so the API is self-documenting.
- Scopes per API key (e.g. execute:protocols, read:status).
- Webhook URL registration per tenant in the DB instead of per-request.
- Database migrations (e.g. node-pg-migrate) instead of CREATE TABLE in code.
- Automated test suite (e.g. Jest + supertest) for regression safety.

Low priority (v3+)
- Pagination for a list endpoint (e.g. GET executions with cursor).
- GET /health for monitoring and load balancers.
- Simulate "rejected" status for testing failure paths.
- Dockerfile for the API so one docker-compose up runs the full stack.
- MCP layer so AI agents can call the API as a tool.
